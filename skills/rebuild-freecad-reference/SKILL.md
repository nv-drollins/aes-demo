# Rebuild an approved reference in FreeCAD

Create editable geometry in a separate FreeCAD document. Derive every coordinate from source Shapes or an approved design parameter; never edit or redraw the reference document.

## Prerequisites

- Require a successful `REFERENCE_AUDIT_OK` result and human approval of the reference views.
- Keep `CliffHouseReference` read-only and use `CliffHouseRebuild` for the current demo.
- Identify the intended deliverable before constructing it: site plan, terrain, massing, walls, openings, or details.

## Procedure

1. Inspect source object metadata and bounds with typed FreeCAD MCP tools.
2. Produce a short semantic mapping: source object, observed type, layer, bounds, proposed role, and confidence.
3. If a curated adapter exists, run it exactly. Do not rewrite it in the prompt.
4. For the current cliff-house site plan, execute this exact code with FreeCAD MCP `execute_code`:

   ```python
   exec(compile(open('/home/nvidia/aec-demo/scripts/build-cliff-house-site-slabs.py', encoding='utf-8').read(), '/home/nvidia/aec-demo/scripts/build-cliff-house-site-slabs.py', 'exec'))
   ```

5. Require `SITE_SLABS_BUILD_OK`. Verify the five outputs and their `SourceReferenceObject`, `SourceRhinoName`, `SourceRhinoLayer`, `MaterialRole`, dimensions, and bounds.
6. Show a fitted Isometric view of only the rebuild document and stop for human approval.
7. If no curated adapter exists for the requested geometry, stop after the mapping and propose a reusable checked script. Do not improvise a long FreeCAD program in chat.

## Construction rules

- Derive, do not redraw: use source wires, vertices, elevations, and stable metadata.
- Put new geometry only in the rebuild document.
- Give every derived object `GeneratedBy`, source identity, and semantic-role properties.
- Use FreeCAD's built-in `Part` and `Mesh` modules. Never use `freeocc` or `cadquery`.
- Make operations idempotent: replace only objects previously generated by the same checked adapter.
- Save a FreeCAD checkpoint after each approved phase.

## Verification

Require the adapter's success marker, typed object inspection, non-zero bounds, source traceability, and a human visual gate. A marker alone is not design approval.
